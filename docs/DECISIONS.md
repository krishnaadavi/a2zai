# Architectural and Product Decisions

## ADR-0001: Product Repositioning

- Date: 2026-02-11
- Status: Accepted
- Decision: Reposition A2Z AI as an intelligence-first product ("Bloomberg for AI builders/investors").
- Rationale: Better differentiation, clearer monetization path, stronger retention loop through tracking behavior.

## ADR-0002: Keep Learn as Secondary Pillar

- Date: 2026-02-11
- Status: Accepted
- Decision: Retain AI Education capabilities and routes; reduce homepage and primary-nav emphasis.
- Rationale: Preserve existing value and SEO while shifting top-level product identity.

## ADR-0003: Priority Sequence Adjustment

- Date: 2026-02-11
- Status: Accepted
- Decision: Prioritize web IA/look-and-feel repositioning before iOS launch completion.
- Rationale: Launching mobile before product clarity risks weak activation and retention.

## ADR-0004: Documentation Operating Model

- Date: 2026-02-11
- Status: Accepted
- Decision: Maintain versioned docs for vision, roadmap, architecture, progress, deployment, integrations, and memory continuity.
- Rationale: Reduce context loss across sessions and improve execution consistency.
